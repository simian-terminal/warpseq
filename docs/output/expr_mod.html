Mod Expressions

Any Literal Notes or Scale Notes or Chords in a Pattern may be followed by a space and then one or more modifiers expressions, separated by spaces.

In basic Patterns, these modify the notes in the pattern in different ways, depending on the expression used. In Transforms, however, these expressions do not include a preceeding note or chord symbol, and modify the currently playing part of the pattern on which they are applied as the clip is evaluated. Read up on Mod Expressions before learning Transforms, and this should be fairly straightforward.

We will start this chapter with some examples of the more basic note modifiers, a few of which were alluded to in the Pattern chapter.

Octave Shifts

A basic example of a Mod Expression shows some octave shifts applied to some notes. Assuming the scale was C major and the base octave was 4... 
1 O+1
1 O-2
2 O=6
3
4
5
... the above set of patterns would produce a C5, a C2, and D6, followed by an unmodified E4, F4, and G4.

Velocity Controls

MIDI velocity can be overriden on individual notes. As a reminder, the default MIDI velocity is set on the Device. 

1 v=100
2 v=50
3 v=120
Velocity takes a value between 0 and 100, and does not always map to volume - it depends on your instrument. See the More Randomness section for more tricks involving MIDI velocity. The maximum velocity value is 127.

MIDI CC Controls

MIDI CCs (Continous Controller messages) can be used to add interesting variations to instruments that support them. Commonly many hardware instruments map MIDI CC1 to the mod wheel, and a variety of other MIDI CCs are usually covered in the reference manual. When using software synths, mapping CCs is usually somewhere in the configuration panels or CCs are hard coded and mentioned in the manual or help pages. 

1 cc1=20
4
10 cc1=80
In the above example, the MIDI CC value for channel 1 is "20" for the first 2 notes, and 80 for the last note. MIDI CC values are sticky. See the More Randomness below for more tricks involving MIDI CCs. CCs can range in value between 0 and 127.

Degree Shifts

Making a note go up or down a scale interval works as follows: 
1 +5
1 -b3
The above example shows moving a note up a perfect fifth and down a flat third. This doesn't make much sense in a basic Pattern, but would make more sense in a Transform. Fair warning: this can select notes outside the current scale. Use "S+" and "S-" if you would like to stay inside the current scale.

Flats and Sharps

Accidentals outside the scale are easy to access: 
1
2
3 b
4
5 #
6
7
Note that sometimes the applied accidentals will still be inside the scale. It depends on the scale. These can also be used, like all mod expressions, in Transforms.

Scale Shifts

Scale shifts can jump a note up or down within the current scale: 
1 S+2
4 S-2
This may seem pointless, because we could have just said "3" or "2" here, but these can be used in powerful ways inside of arpeggiators built with Transforms, or with Variables. See more examples on those pages.

Probability Events

A probability event can be assigned so that the very next mod expression has only a partial chance of firing. 
1 p=0.5 O+2
In the above example, the note will jump an octave only half of the times the pattern plays. These are a form of randomness, so this section is somewhat of a spoiler for the More Randomness section.

Silence Events

Silences are most useful with probabilities. 
1 p=0.5 x
In the above example, the event will not play half of the time. This has some relation to the Rests & Ties section.

Chordification

While chords can be explicitly entered, chords can also be created from mod expressions in patterns or in arpeggiators. See the "Randomness" section below for more tricks involving chords. 
1 ch=power
4 ch=major
3 ch=sus4

Combining Mod Expressions

As shown in a few examples above, multiple mod expressions can be added to the same line: 
3 b ch=power O+2
4
5

This first note takes the 3rd note in the current scale, applies a flat, converts the note into a power chord, and transposes the chord up one octave. Most combinations should have the same effect regardless of what order the expressions are applied.
