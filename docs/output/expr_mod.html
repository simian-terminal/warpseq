Mod Expressions

Any literal notes or scale notes in a Pattern may be followed by a space and one or more modifiers, separated by spaces.

In basic Patterns, these modify the notes in the pattern. In Transforms, however, these expressions do not include a preceeding note or chord symbol, and modify the currently playing part of the pattern on which they are applied as the clip is evaluated.

We will start this chapter with some of the more basic note modifiers, shown in Pattern examples.

Octave Shifts

A basic example shows some octave offsets. Assuming the scale was C major and the base octave was 4... 
1 O+1
1 O-2
2 O=4
... the above set of patterns would produce a C5, a C2, and D4.

Velocity Controls

Velocity can be adjusted in one of many ways. 
1 v=100
2 v+10
3 v-10
Velocity takes a value between 0 and 100, and does not always map to volume - it depends on your instrument. The latter two examples might be interesting in arpeggiator techniques with Transforms. See the More Randomness section for more tricks involving MIDI velocity.

MIDI CC Controls

MIDI CCs can be used to add interesting variations. 
1 cc1=20
4
10 cc1=80
In the above example, the MIDI CC value for channel 1 is "20" for the first 2 notes, and 80 for the last note. MIDI CCs are sticky. See the More Randomness below for more tricks involving MIDI CCs.

Degree Shifts

Making a note go up or down a scale interval works as follows: 
1 +5
1 -b3
The above example shows moving a note up a perfect fifth and down a flat third. This doesn't make much sense in a basic Pattern, but would make more sense in a Transform. Fair warning: this can select notes outside the current scale.

Flats and Sharps

Accidentals outside the scale are easy to access: 
1
2
3 b
4
5 #
6
7
Note that sometimes the accidentals will still be inside the scale. It depends on the scale. These can also be used, like all mod expressions, in Transforms.

Scale Shifts

Scale shifts can jump a note up or down in the current scale: 
1 S+2
4 S-2
This may seem pointless, because we could have just said "3" or "2" here, but these can be used in powerful ways inside of arpeggiators built with Transforms.
  REMINDER: We do not have different docs for mod expressions - the parts that come after the notes in Patterns, and mod expressions as used in Transforms. They are usable everywhere. Transforms just don't have the preceeding note information.

Probability Events

A probability event can be assigned so that the next event has only a partial chance of firing. 
1 p=0.5;O+2
In the above example, the note will jump an octave only half of the times the pattern plays. These are a form of randomness, so this section is somewhat of a spoiler for the More Randomness section.

Silence Events

Silences are most useful with probabilities. 
1 p=0.5;x
In the above example, the event will not play half of the time. This has some relation to the Rests & Ties section.

Chordification

While chords can be explicitly entered, chords can also be created from mod expressions in patterns or in arpeggiators. See the "Randomness" section below for more tricks involving chords. 
1 ch=power
4 ch=major
3 ch=sus4
