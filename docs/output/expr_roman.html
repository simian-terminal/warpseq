Scale Notes & Chords

Input of notes that follow Scale settings within Warp follow a varient of something called "Roman Numeral Analysis". In short, Warp uses arabic numerals to indicate degrees along a scale, and roman numerals to represent basic chords. Jazz fans will object to this, and say this can't possibly represent all the types of interesting and useful chords, and they are correct - but bear with us.

A Scale Example

Here is the same scale mentioned in Literal Notes in a way that will work for any 7-note Scale setting: 
 1
2
3
4
5
6
7
8
This means that we can define a Pattern and then the pattern can be then used in any clip using any scale we want.
  Most other sequencer programs and DAWs do not natively understand scales, but instead "force" notes to the nearest scale note. This means multiple note entries might map to the same note, or it may be unclear if something rounds up or down. Of greater concern, in these force-to-scale systems, the ability to use accidentals is lost.

Roman Numeral Chords

Chords can be expressed as Roman numerals: 

I
IV
V
This notation is commonly used to denote chord progressions. Like the arabic numerals above, it is nice because it doesn't care what scale you are in. Lowercase roman numerals represent minor chords: 

i
iv
v
More advanced types can be represented using their base notes and chord type: 

1 ch=sus4
2 ch=major
4 ch=power
This is a bit of a spoiler as this refers to "Mod Expressions", covered immediately below. For reference the built-in chord types you can use this way are:
  • minor
  • major
  • dim
  • aug
  • sus4
  • sus2
  • fourth
  • power
  • fifth
  • M6
  • m6
  • dom7
  • M7
  • m7
  • aug7
  • dim7
  • mM7
That's a little jazzier, we just don't get to use the roman numeral syntax.
