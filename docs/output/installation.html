Installation Hello

Requirements

So far, Warp has been tested on Mac and Linux. If you are on Windows, we would appreciate help in testing. First install python3 and git if you do not already have it. On a Mac, we recommend getting Python3 from homebrew: 

brew install python3
brew install git

Otherwise, do what your platform needs you to do to install Python 3 and git.

Checkout

Now clone the project into a directory: 

cd wherever
git clone https://laserllama@bitbucket.org/laserllama/warpseq.git

Libraries

Now install the Python dependencies: 

pip3 install -f requirements.txt
  Fun fact: Warp uses a python library called classforge, also from Michael DeHaan. The documentation for Warp was generated with Michael's project rockchisel. Python developers may be interested in these.

Creating a Virtual Bus

If you want to use Warp with software (as opposed to hardware) synthesizers, we need to create a Virtual Bus to talk to them. Ableton has already written some great documentation on this, so we suggest following those:

How To Setup A Virtual MIDI Bus [Ableton.com].

(You can skip step 5 unless you also use Ableton and haven't already set that up)

Ready?

We are ready to go. At the current state of development, only the python API is available. The UI is coming soon. To try out Warpseq from the API, run one of the examples: 


python3 tests/api.py

Except this won't probably work, because you need to select your MIDI interface! Hack the example for now where you see "Device" to match the name of a device you have. FIXME: this instruction will change.

When the UI is available, there will just be a few more install steps.
