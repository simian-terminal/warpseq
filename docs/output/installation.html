Installation

Thank you for interest in trying out Warp!
  Warp is in a API-only beta phase through Fall 2020 as we work to build the performance-oriented user interface. The instructions below cover setting up the API and will include install instructions for the UI once released. For now, we will assume some basic Python development and command line skills. If you know another programming language, you should find Python easy to learn, and Warp requires only a small amount of Python knowledge.

Requirements

Warp should work on most any computer. First install python3 and git if you do not already have them. On a Mac, we recommend installing these from homebrew: 


brew install python3
brew install git

Checkout

Now clone the project into a directory: 

cd wherever
git clone https://laserllama@bitbucket.org/laserllama/warpseq.git

Libraries

Now install the dependencies: 

pip3 install -r requirements.txt
  If the pip install fails on a Mac, you may need to install xcode developer tools, run: "xcode-select --install"

Creating a Virtual Bus

If you want to use Warp with software synthesizers, you will need to create a Virtual Bus in your operating system if you do not already have one. Ableton has already written some great documentation on creating them:

How To Setup A Virtual MIDI Bus [Ableton.com].

You can skip step 5 in the above document -- it's Ableton specific. Further, if you want to just use hardware, and don't need to route MIDI through your DAW, you can also choose to not set up a virtual bus.

  When writing your tracks later, unless you feel the need to record MIDI notes vs audio, you should NOT route MIDI through the IAC bus into your software synthesizers. You will have better performance talking to your MIDI interface hardware directly. You will also have the added bonus of not having to "arm" MIDI tracks to test your compositions.

Ready?

We are ready to go. At this point of the Warp beta, only the python API is available. The UI is coming soon in the Fall of 2020. To try out Warpseq from the API, we will use python to run one of the examples: 

cd examples/api
python3 tests/00_kick_snare.py

The program may prompt you about selecting a MIDI device name. If the demo sounds strange, you probably need to choose some different instruments.

When the UI is available, there will just be a few more install steps for Web UI dependencies.

Next Steps

With everything installed, you are ready to try some more of the Python API examples. However we also recommend you read through the "Concepts" and "Expression Language" sections (on the left) to get a better feel of what the examples are teaching.

The API examples are only the start, what you create is up to your imagination and can get rather advanced as you start incorporating concepts like "transforms". Have fun trying out the examples and exploring the docs!

If you get stuck or have a question, we're trying to refine our documentation at this stage so please Email Michael. We'd be glad to hear from you!
