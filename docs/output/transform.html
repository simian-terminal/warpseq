Transforms

Transforms are like meta-patterns that serve as MIDI effects. They process patterns and modify the result before they are turned into MIDI events. Transforms take notes listed in patterns inside a Clip and then do "things" to them - they don't just have to change the notes - for instance, they can also change velocity and MIDI CC data. Using just a few transforms can add a lot of sonic variety to repeated patterns.

  • Classically, arpeggiators are used in computer synthesizers to turn chord patterns into individual notes, played in rapid sequence
  • In Warp, transforms CAN be used to create arpeggiators, but that is only the most basic usage of transforms.
  • in Warp, transforms can also run against individual notes (not just chords), and even turn those notes *back* into chords.
    • .
  • Transforms achieve this power through Mod Expressions.
  • Transforms have a divide time that decides how much they slice up notes or chords. For processing triads, usually use will want to set this to a multiple of 3.
  • in Warp, a Clip can contain a list of transforms, where one or more transforms is applied in rotation after another.

Examples

Transform examples are included as part of the Expression Guide: see here. Reading this chapter will give a lot better understanding of how Transforms work and what they can do. The API also has some very good transform examples. Once the UI is available, the demo songs will also include some interesting transform examples.
